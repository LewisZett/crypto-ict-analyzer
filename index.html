ICT Price Action Analysis App

Pick a market below. Crypto screens the CoinGecko top 50 by market cap with candles from Binance; Forex screens your configured pair list with quotes and candles from Twelve Data. Either way, the same local filter (moving average, market structure, Fibonacci OTE, liquidity sweep, FVG) computed in JavaScript decides what's even worth asking about, then Gemini AI judges context/session quality only for candidates that already pass the hard technical rules.

Security: Your Gemini API key is sent directly from your browser and is never stored. Use only on a trusted device.

Settings (risk, thresholds, timezone)
Liquidation mode: TP is still placed using the structural SL/RR math above (that's what decides where you take profit), but the trade is only counted as a loss if price reaches your real liquidation price - not the tight structural SL. The R-multiple shown is rebased against liquidation distance (your real risk unit), so a win at TP will show as a much smaller R than the "targetRR" setting - that's expected, since TP is close relative to how far liquidation is.
Confluence Scoring (replaces the old all-or-nothing strict filter)
Each factor below contributes points if it's present. A trade needs a total score ≥ the threshold to pass - it no longer needs every single factor at once. One hard rule stays outside the score: if BOTH the MA trend and the structure (HH/HL or LH/LL) disagree with the trade direction, it's rejected regardless of score - this stops outright counter-trend trades from sneaking through on sweep/FVG/volume alone.
Backtest (no API key needed) - tests the local filter logic against recent historical data

This does NOT call Gemini - it only replays the deterministic MA/structure/OTE/SL/TP rules against past candles, so you can see how the filter itself would have performed before trusting any of its live signals.

Settings Sweep (no API key needed) - tests many symbols, RR values, confluence thresholds, and SL buffer sizes in one run

Runs the same deterministic local-filter logic as the single backtest above, but fetches candles for every symbol you list ONCE and then replays every RR value, every minimum confluence score, AND every SL buffer size you list against those same trades - so instead of guessing at a threshold, you get a grid comparing every combination side-by-side on a real sample size. Confluence weights come from the Settings panel above; this sweeps the pass/fail threshold and stop placement, not the weights themselves.

Trade Journal - every recommendation from a live run is saved here automatically

Nothing here is graded until you click "Check Outcomes" - that pulls the real candles that happened after each trade was logged and checks whether price actually hit TP or SL.
